# T-020: Normalize date columns and sorting

## Goal

- Render the `Start` and `Registration deadline` columns in the same visible date format.
- Make both table sorts use actual date values instead of raw strings.

## Scope

- Added shared helpers in `assets/app.js` for effective registration deadlines, date parsing, and visible `dd.mm.yyyy` formatting.
- Updated the `Start` column renderer to use the same formatted date output as the registration deadline column.
- Switched date sorting to parsed timestamps and made the registration deadline column sort by `details.registration_until || row.end`.
- Kept empty dates at the bottom of the table instead of mixing them with real values during sorting.

## Verification

- `node --check assets/app.js`
- Reviewed the `compare()` logic to confirm that both `start` and effective registration deadline sorting paths now use parsed timestamps.
