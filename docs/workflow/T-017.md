# T-017: Narrow lottery table and merge venue into requirements

## Goal

- Reduce the visible width of the published lottery table so it fits the page more comfortably, remove the dedicated venue column, and preserve the most useful venue context inside the requirements column.

## Scope

- Remove the `place` column from the table header and row rendering.
- Keep the permit column compact because permit numbers are consistently short.
- Update the requirements cell rendering so it can include useful venue context from the registry row without losing the existing enrichment text.
- Tighten table sizing and column widths in `assets/style.css` so the table requires less horizontal space on desktop and mobile.
- Keep search and sorting behavior intact for the remaining visible columns.

## Verification

- `python -m json.tool assets/active_lotteries.json`
- `python -m json.tool assets/enriched_info.json`

## Implemented

- Removed the dedicated `place` column from the published table in `index.html`.
- Reduced the table from `10` visible columns to `9` and updated the loading/error/empty-state `colspan` values to match.
- Added a `buildRequirementsCell()` renderer in `assets/app.js` that combines:
  - the existing enriched participation requirements,
  - the registry venue text prefixed as `Где действует:`,
  - the registration deadline prefixed as `Регистрация до:` when present.
- Kept venue text searchable by preserving `row.place` in the search haystack even though the dedicated column is no longer rendered.
- Tightened the table layout in `assets/style.css`:
  - lowered the base `min-width`,
  - set a compact fixed width for the permit column,
  - reduced widths for date and registration columns,
  - constrained the remaining columns so the table fits the viewport more comfortably.

## Final Verification

- `python -m json.tool assets/active_lotteries.json`
- `python -m json.tool assets/enriched_info.json`
- Manual structure check confirmed a consistent 9-column table between the header, row rendering, and fallback states.
