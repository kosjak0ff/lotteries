# T-021: Clarify Mego brands in requirements table

## Goal

- Make the published Mego lottery row for permit `8943` show the qualifying participating brands explicitly in the requirements column.
- Keep the client-side table search aligned with the new brand metadata.

## Scope

- Added an `eligible_brands` field to `assets/enriched_info.json` for permit `8943`, using Appendix No. 2 from the official `rules_8943.pdf` as the source of truth.
- Updated `assets/app.js` so the requirements cell renders a dedicated `Подходят бренды:` line when enriched data includes `eligible_brands`.
- Extended the table search haystack to include `details.eligible_brands`, so searches by participating brand names find the Mego row.
