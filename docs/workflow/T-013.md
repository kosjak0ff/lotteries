# T-013: Deduplicate IAUI update issues across history

## Goal

- Prevent duplicate `iaui-update` GitHub issues when the IAUI page still exposes the same metadata fingerprint as an earlier notification.

## Scope

- Update the GitHub Actions notification workflow to deduplicate against all historical `iaui-update` issues instead of only open ones.
- Reuse an existing issue for the same fingerprint, reopening it when needed, and close any stale open notification issues for other fingerprints.

## Implemented

- Updated `.github/workflows/check-iaui-update.yml` so the workflow loads `iaui-update` issues with `state: "all"`.
- Added reuse logic for a matching `fingerprint`:
  the workflow now reopens the existing issue if it was closed, closes any other open notification issues, and exits without creating a duplicate issue.
- Preserved the existing behavior for genuinely new metadata fingerprints:
  older open alerts are closed and a new notification issue is created.

## Verification

- `python -m py_compile scripts/check_iaui_update.py`
- `python scripts/agentctl.py verify T-013`
