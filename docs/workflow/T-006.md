# T-006: Notify on IAUI file updates

## Goal

- Detect changes in the published IAUI lottery registry file metadata and notify via GitHub Issue without changing site data.

## Scope

- Add a lightweight IAUI metadata check script.
- Add a scheduled/manual GitHub Actions workflow.
- Use GitHub Issues as the notification surface and deduplication state.

## Implemented

- Added `scripts/check_iaui_update.py` to read the IAUI page, discover the current download URL, extract the `Atjaunināts` date, and compute a stable metadata fingerprint.
- Added `.github/workflows/check-iaui-update.yml` with both `workflow_dispatch` and a daily scheduled run.
- Configured the workflow to use GitHub Issues as the notification mechanism:
  it ensures the `iaui-update` label exists, deduplicates by fingerprint, closes older open notification issues, and opens a fresh issue only when the published file metadata changes.
- Kept the site data untouched: no XLS download into the repo, no dataset regeneration, and no bot commits for content updates.

## Verification

- `python -m py_compile scripts/check_iaui_update.py`
- `test -f .github/workflows/check-iaui-update.yml`
