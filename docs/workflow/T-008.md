# T-008: Remove search hint and make title dynamic

## Goal

- Remove the static search hint text from the search controls.
- Make the browser page title follow the current dataset reference date instead of a hardcoded date.

## Scope

- Update the static HTML markup so the search hint is no longer shown by default.
- Update the client script so the title is derived from `payload.reference_date`.
- Keep the visible reference date in the hero synced with the same data source.

## Implemented

- Removed the `#search-hint` node from `index.html`, so the controls no longer render the static helper text.
- Replaced the hardcoded `<title>` date with a generic base title in `index.html`.
- Added `formatReferenceDate()` in `assets/app.js` to convert `YYYY-MM-DD` payload dates into `DD.MM.YYYY`.
- Updated the bootstrap flow to set both the visible hero date and `document.title` from `payload.reference_date`.
- Kept a safe fallback so the page title stays generic if the dataset does not provide a reference date.

## Verification

- `python scripts/agentctl.py verify T-008`
- Confirmed `assets/active_lotteries.json` currently exposes `reference_date: 2026-03-23`.
- Confirmed the computed browser title now resolves to `Активные лотереи Латвии · 23.03.2026`.
- Confirmed the removed search hint text no longer exists in `index.html` or `assets/app.js`.
