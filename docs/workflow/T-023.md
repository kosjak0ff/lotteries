# T-023: Harden IAUI file change monitoring

## Goal

- Improve the daily IAUI/VID lottery registry monitor so it detects real XLS payload changes, avoids unrelated fallback downloads, and remains stable when the page markup shifts.

## Scope

- Update `scripts/check_iaui_update.py` to identify the intended lottery registry link more strictly.
- Extend the metadata payload/fingerprint with file-level change detection derived from the published XLS response.
- Add deterministic automated tests for link discovery, metadata extraction, and fingerprint behavior.
- Adjust the GitHub Actions workflow only if the updated metadata contract requires it.

## Implemented

- Tightened `scripts/check_iaui_update.py` so it only accepts the intended lottery registry download link instead of falling back to the first `/media/` or `download?attachment` link on the page.
- Added file-level change detection by downloading the published XLS, computing its SHA-256 hash, and including that hash in the emitted metadata payload and stable fingerprint.
- Exposed `file_sha256` and `file_size` in the JSON output for easier debugging and verification of detected changes.
- Updated `.github/workflows/check-iaui-update.yml` so notification issues include the file SHA-256 alongside the existing metadata summary.
- Added `tests/test_check_iaui_update.py` with deterministic `unittest` coverage for target-link selection, rejection of unrelated downloads, `Atjaunināts` extraction, and fingerprint changes when the file hash changes.

## Verification

- `python -m py_compile scripts/check_iaui_update.py`
- `python -m unittest discover -s tests -p "test_*.py"`
- `python scripts/check_iaui_update.py --output /tmp/iaui-metadata.json`
