# T-014: Fix IAUI update workflow failure

## Goal

- Investigate why the `Check IAUI Update` GitHub Actions workflow failed on `main` for run context `3c48564` and restore a successful notification flow without breaking the existing issue deduplication behavior.

## Scope

- Inspect the failing workflow run logs to isolate whether the breakage comes from `scripts/check_iaui_update.py`, the `actions/github-script` step, or GitHub API assumptions.
- Apply the smallest safe fix in the workflow helper script.
- Preserve the historical `iaui-update` issue reuse logic introduced by T-013.

## Findings

- The failed scheduled runs on `2026-04-02` and `2026-04-03` both stopped in `python scripts/check_iaui_update.py --output iaui-metadata.json` with `RuntimeError: Could not find a download link on the IAUI page`.
- The old default source URL no longer exposes the IAUI registry page content. It now resolves to a VID page shell after the IAUI-to-VID migration, so the old regex-based parser never sees the expected download link or `Atjaunināts` text.
- The current official registry source is `https://www.vid.gov.lv/lv/loterijas`, which contains the `Izsniegtās preču un pakalpojumu loteriju atļaujas` file link and the page metadata update date.

## Implemented

- Updated `scripts/check_iaui_update.py` to use `https://www.vid.gov.lv/lv/loterijas` as the default page URL.
- Replaced the fragile regex-only anchor search with a small `html.parser.HTMLParser`-based link extractor that reads link text plus `aria-label` and `title` attributes.
- Kept the existing fingerprint contract (`download_url`, `link_text`, `updated_at`) intact so the GitHub Actions issue deduplication logic in `.github/workflows/check-iaui-update.yml` continues to work unchanged.
- Stored the final resolved page URL in the emitted JSON so redirects no longer produce misleading `page_url` values.

## Verification

- `python -m py_compile scripts/check_iaui_update.py`
- `python scripts/check_iaui_update.py --output /tmp/iaui-metadata.json`
- `python scripts/agentctl.py verify T-014`
- Local output now resolves to:
  `page_url=https://www.vid.gov.lv/lv/loterijas`
  `download_url=https://www.vid.gov.lv/lv/media/34040/download?attachment`
  `updated_at=26.03.2026`
