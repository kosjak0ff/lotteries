# T-015: Remove agent workflow files from GitHub

## Goal

- Stop publishing the local agent workflow files to GitHub while keeping the same files available in the local development workspace.

## Scope

- Remove the agent workflow files from git tracking without deleting them from disk.
- Add ignore rules so the removed files do not immediately reappear as local untracked noise.
- Limit the change to the agent-specific workflow files and their tracking state only.

## Implemented

- Added ignore rules for `AGENTS.md`, `.AGENTS/`, `tasks.json`, `docs/agentctl.md`, `docs/workflow/`, and `scripts/agentctl.py`.
- Removed those same paths from git tracking with `git rm --cached -r ...`, which keeps the files in the local working tree while deleting them from the repository history going forward.
- Left the rest of the repository unchanged so local development can continue to use the existing agent files from disk.

## Verification

- Confirm the targeted agent workflow paths are no longer returned by `git ls-files`.
- Confirm the same files still exist locally after the change.
- `git ls-files AGENTS.md .AGENTS docs/workflow docs/agentctl.md scripts/agentctl.py tasks.json` now prints nothing.
- `test -f AGENTS.md && test -f tasks.json && test -f scripts/agentctl.py && test -d .AGENTS && test -d docs/workflow` passes locally.
